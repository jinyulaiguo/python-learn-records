## 3. 脚本使用说明与参数传参形式

本脚本采用了健壮的 **Bash 参数扩展语法 (`${VAR:-default}`)**，支持从零参数到多参数的灵活调用。

### 形式一：零参数传入（全自动默认模式）
如果你只是想快速建个测试项目，不想思考命名：
```bash
./create_agent.sh
```
* **执行结果**：
  * 项目文件夹名：`my-ai-agent` （触发 `$1` 的默认值）
  * 核心 Python 包名：`my_ai_agent` （自动将横杠替换为下划线）

### 形式二：传入一个参数（最常用：指定项目名）
这是日常开发中最推荐的做法。你只需定义项目目录的名称（推荐使用连字符 `kebab-case`）。
```bash
./create_agent.sh deepseek-financial-agent
```
* **执行结果**：
  * 项目文件夹名：`deepseek-financial-agent` （被赋值给 `$1`）
  * 核心 Python 包名：`deepseek_financial_agent` （脚本自动推导，触发 `${PROJECT_DIR//-/_}` 逻辑）

### 形式三：传入两个参数（自定义项目名与包名）
极少数情况下，当你希望项目文件夹的名称和内部代码包的名称完全不相关时（不推荐，但脚本支持）：
```bash
./create_agent.sh backend-service core_ai_logic
```
* **执行结果**：
  * 项目文件夹名：`backend-service` （被赋值给 `$1`）
  * 核心 Python 包名：`core_ai_logic` （被强制赋值给 `$2`，覆盖了自动推导逻辑）
## 4.项目结构
```Plaintext
my-ai-agent/
├── .env                 # 本地环境变量配置（绝不能提交到 Git）
├── .env.example         # 环境变量模板（供团队其他成员参考）
├── .gitignore
├── pyproject.toml       # 现代 Python 项目的核心配置文件（类似 pom.xml）
├── uv.lock              # 依赖版本锁定文件

│
├── src/                 # 核心代码目录（采用 src-layout 最佳实践）
│   └── my_agent_project/    # 你的包名（使用下划线）
│       ├── __init__.py      # 标识这是一个 Python 包
│       │
│       ├── api/             # 接入层：负责处理 HTTP 请求与响应（类似 Controllers）
│       │   ├── __init__.py
│       │   ├── dependencies.py # API 依赖注入（如认证、数据库 Session）
│       │   └── routes/         # 具体的路由文件
│       │       └── chat.py     # 例如：暴露给前端的对话接口
│       │
│       ├── core/            # 核心配置层
│       │   ├── __init__.py
│       │   ├── config.py       # 全局配置管理（推荐使用 Pydantic Settings）
│       │   └── exceptions.py   # 全局自定义异常处理

│       ├── agents/          # AI 业务逻辑层（项目核心！）
│       │   ├── __init__.py
│       │   ├── state.py        # 定义 LangGraph 的 State（状态流转的数据结构）
│       │   ├── nodes.py        # LangGraph 的各个节点逻辑（比如调用 DeepSeek、查数据库）
│       │   ├── tools.py        # Agent 可调用的外部工具（Function Calling）
│       │   ├── llm.py          # 封装 DeepSeek 模型的初始化与基础调用

│       ├── services/        # 传统业务逻辑层（类似 Service 层）
│       │   ├── __init__.py

│       │
│       ├── models/          # 数据模型层
│       │   ├── __init__.py
│       │   ├── domain.py       # 数据库 ORM 模型（如 SQLAlchemy / SQLModel）
│       │   └── schemas.py      # 接口请求/响应的验证模型（Pydantic / 类似 DTO）
│       │
│       └── utils/           # 通用工具类
│           ├── __init__.py
│           └── logger.py       # 结构化日志配置
│
└── tests/               # 单元测试与集成测试
    ├── conftest.py      # Pytest 的全局 fixture 配置

    └── test_agents/     # 针对 LangGraph 节点的独立测试
```
